# ADR-005: Role-Based Execution Model

## Status
Accepted

## Context
AI tasks need structure and boundaries.

## Decision
All AI actions occur through defined roles (Planner, Sprint Controller).

## Rationale
Encapsulates behavior and prevents uncontrolled execution.

## Alternatives Considered
- Free-form AI execution

## Consequences
### Positive
- Predictable behavior

### Negative
- Requires role definition effort
